# M10 Integrate factory and escrow status into the indexer

## Outcome
- Added a dedicated spawn-session/backend surface in the indexer with `apps/indexer/src/routes/spawn-sessions.ts` for normalized session reads and spawned-automaton registry discovery.
- Added `apps/indexer/src/integrations/factory-client.ts` and `apps/indexer/src/integrations/escrow-client.ts` as injectable normalization seams that merge factory session/audit data with escrow payment state.
- Extended the SQLite store and schema to cache spawn sessions and spawned-automaton registry records separately from the public automaton list.
- Extended realtime filtering so websocket subscribers can scope to `sessionId` without changing the existing automaton event path.
- Added shared spawn-session detail/escrow contracts plus indexer tests that verify session caching stays separate from `/api/automatons`.

## Learnings
- The cleanest way to keep session status separate from the public automaton list was to add dedicated `spawn_sessions` and `spawned_automaton_registry` tables instead of overloading the existing automaton snapshot tables.
- Workspace validation needed split TypeScript resolution: the indexer lint/test config now resolves `@ic-automaton/shared` from source, while the indexer build config resolves the built shared declarations so `rootDir` stays valid during `npm run build`.
- The backend route can already normalize factory session data with escrow state, but the transport is intentionally injectable; no hard-coded IC/escrow wire protocol was introduced in this milestone.
