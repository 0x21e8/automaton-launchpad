## Learnings
- The prior npm-registry blocker was stale for this workspace: `@fastify/websocket` and `better-sqlite3` were already present in `package-lock.json` and `node_modules`, so the milestone could be completed without adding substitute implementations.
- Fastify's full route declaration with `handler` plus `wsHandler` lets the realtime endpoint preserve the existing HTTP `426 Upgrade Required` response while also supporting actual websocket upgrades on the same path.
- `better-sqlite3` does not ship usable local TypeScript declarations in this workspace, so the store uses a small typed `createRequire()` boundary rather than reintroducing a shell-based database path.
