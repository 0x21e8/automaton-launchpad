# M07 Implement factory session core

## Outcome
- Replaced the factory bootstrap stub with a milestone-scoped Rust module layout covering `types`, upgrade-safe state storage, public APIs, and admin APIs.
- Implemented fixed quote generation for `ETH` and `USDC`, deterministic factory-issued `sessionId` creation, immutable `quoteTermsHash` computation, session expiration timestamps, and normalized payment instructions on session creation.
- Added coarse global pause handling, admin quote/config reads and updates, canonical registry read methods, and per-session audit-log plumbing with snapshot/restore tests for later canister upgrade wiring.

## Learnings
- The current offline workspace has no IC canister runtime crates wired in, so upgrade-safe persistence is expressed as `FactoryStateSnapshot` plus `snapshot_state`/`restore_state`; later milestones can bind that seam to real pre-upgrade/post-upgrade hooks without redesigning the domain model.
- Fixed quote config is modeled as one global admin-managed config with asset-specific amounts for `ETH` and `USDC`, which keeps the v1 fee model fixed while still producing deterministic per-asset quotes.
- `create_spawn_session` rejects quoted gross amounts below the configured fee-plus-creation minimum; later escrow milestones should treat underfunding as “paid less than the locked quote,” not “created an invalid quote.”
