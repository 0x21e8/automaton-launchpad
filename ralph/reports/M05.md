# M05 Implement spawn wizard and provider configuration UX

## Outcome
- Built the six-step spawn wizard in `apps/web` with local/sessionless state for chain, risk, strategies, skills, provider configuration, and funding.
- Added provider configuration UX that prefers live OpenRouter model loading and falls back to a curated local model list when the request fails or is unavailable.
- Added funding-step gross-payment validation against the locked `$50` minimum and disclosed gross payment, platform fee, creation cost, and net forwarded amount.
- Wired the wizard into the existing shell through the `Spawn` nav action and added focused web tests for the wizard shell and funding math.

## Learnings
- The repo still intentionally imports shared TypeScript source directly from `packages/shared/src/` inside `apps/web`; the spawn wizard follows that pattern so web lint/test/build do not depend on a prebuilt shared `dist/`.
- The milestone stays sessionless by using fixed local quote terms (`platform fee`, `creation cost`, ETH/USD conversion) inside `spawn-state.ts`; later milestones should replace that with real session/quote data instead of forking the UX copy or math.
- OpenRouter model loading is best-effort in the browser and must degrade cleanly to `apps/web/src/lib/default-models.ts`; the provider step already assumes network failure is normal.
