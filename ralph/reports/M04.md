# M04 Implement grid and detail drawer experience

## Outcome
- Replaced the M03 static viewport placeholder with a canvas-based automaton grid that renders mock organisms, deterministic positions, parent-child links, animated message routes, owned-label behavior, and hover/click interactions.
- Added the bottom drawer shell with shared-contract-backed identity, financial, runtime, monologue, and command-line sections, including steward-only command execution preview plus read-only fallback copy.
- Introduced local mock automaton detail data so the frontend can exercise the M04 experience without wiring the real indexer or backend yet.

## Learnings
- The shared `AutomatonDetail` contract is narrower than `AutomatonRecord`; mock frontend detail payloads must keep runtime and financial extras nested under `runtime` and `financials` instead of copying record-level fields onto the detail root.
- Cross-package frontend imports can safely consume the shared TypeScript source files directly from `packages/shared/src/` in the current workspace layout, which avoids depending on a prebuilt `packages/shared/dist` during app linting.
- The drawer shell is now stable enough for later data wiring: later milestones can swap mock detail/session data for real API results without changing the component boundaries introduced here.
